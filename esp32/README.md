# ESP32-S3 firmware (scaffold)

On-device ESC-50 classifier for the ESP32-S3 (N16R8). Everything that can be
built and verified without the board is now done: the inference graph, model
embedding, quantization I/O, config, dashboard bridge, and the mel front end
(implemented + off-board validated, not just stubbed — see below).

## Layout
- `platformio.ini` — board = `esp32-s3-devkitc-1`, PSRAM on, TFLite Micro +
  WebSockets deps.
- `src/config.h` — **all audio/model params in one place**, must match training
  (`scripts/common.py`): SR 22050, 64 mels, 5s, 216 frames. Also I2S pins + WiFi.
- `src/main.cpp` — TFLite Micro setup, int8 quantize using the tensor's own
  scale/zero-point, top-1 decode, latency print. Runs on a zero input during
  bring-up so you can confirm the model loads before wiring the mic.
- `src/mel_frontend.{h,cc}` — on-device log-mel front end: framing (librosa
  `center=True`/zero-pad semantics), periodic Hann window, FFT, mel filterbank,
  `power_to_db(ref=max, top_db=80)`, per-clip z-norm. Matches
  `scripts/common.py::wav_to_mel` — see validation below.
- `src/fft.{h,c}` — portable radix-2 complex FFT, pure C, no esp-dsp dependency.
  Swapping in esp-dsp's `dsps_fft2r_fc32` later is a pure speed optimization
  (same radix-2 DIT math, hardware-accelerated butterflies), not a correctness
  fix — do it only if profiling on the board shows FFT time is a bottleneck.
- `src/mel_filterbank_data.h` — the 64 triangular mel filters as a sparse
  per-filter (start_bin, weights[]) table (~8.5KB vs ~262KB dense). Generated
  by `scripts/gen_mel_filterbank.py` from `librosa.filters.mel` — do not
  hand-edit; regenerate if SR/N_FFT/N_MELS ever change.
- `src/esc50_model/model_data.{cc,h}` — the model as a C byte array, generated by
  `scripts/export_esp32.py` (`.cc` is gitignored — regenerate it).
- `host_test/harness.cc` — native (no ESP-IDF) driver for `mel_frontend.cc`,
  used by `scripts/validate_mel_frontend.py`.

## Generate the model array
The TFLite export hang (DECISIONS.md M4) is resolved — see DECISIONS.md
"Milestone 4 RESOLVED". Use the clean int8 tflite:
```bash
../esc50env/bin/python3 ../scripts/export_esp32.py --tflite ../tflite_models/mid_student_int8_clean.tflite
```

## Validate the mel front end (no board needed)
```bash
../esc50env/bin/python3 ../scripts/validate_mel_frontend.py
```
Compiles `mel_frontend.cc` natively, runs it on synthetic PCM (tones, noise,
a transient, a chirp), and diffs the output against `common.wav_to_mel` via
librosa. Passing this proves the DSP math is right; it does NOT prove the
real I2S/ADC mic capture is right — that still needs the board. Re-run after
any change to `mel_frontend.cc` or the filterbank.

## Build / flash
PlatformIO lives in `../../pioenv` (py3.11). Flash over the DevKitC **UART** port
(`/dev/cu.usbserial-*`), not the native-USB port — `Serial` is on UART0.
```bash
export PATH="$PWD/../../pioenv/bin:$PATH"
pio run                                              # build (default: self-test mode)
pio run -t upload --upload-port /dev/cu.usbserial-XXXX
pio device monitor -b 115200
```

## Build modes (`-DBRINGUP_MODE`)
- `2` **self-test** (default): runs the full mel→inference pipeline on real ESC-50
  clips embedded in `test_clips.h`. Proves the on-device DSP on real audio without
  the mic. Prints PASS/FAIL + latency and a machine-readable `RESULT {json}` line.
- `1` **model-check**: inference on a zero input — just proves the graph loads and
  gives arena/latency.
- `0` **mic**: live I2S capture → classify → dashboard. Written, but needs the mic
  physically wired and the `>>14` gain shift calibrated on-board.

## Measured on-device (ESP32-S3 N16R8) — see DECISIONS.md M9
- Tiny INT8 (deployed): arena 137 KB (internal SRAM), inference 6.1 s, `dog` +
  `church_bells` classify correctly on real audio (2/2).
- Mid INT8: arena 273 KB — only fits PSRAM, 22.5 s. Tiny is the deployable model.
- Latency is reference-kernel bound; esp-nn (`esp-tflite-micro`) is the next step.

## What genuinely still needs the board
1. **Wire the INMP441 I2S mic** into `mel_frontend_compute()` (mode 0). Real ADC
   noise / I2S timing / gain staging are the only things the host harness and the
   embedded-clip self-test can't exercise. Compare a captured clip's on-device mel
   to `common.wav_to_mel` of the same source before trusting live predictions.
2. **esp-nn kernel swap** for real-time latency (optional, big win).

Model loading, int8 I/O, the mel front end on real audio, RAM/latency, and the
dashboard bridge are all verified on hardware.
